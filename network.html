Chapter 4 Terms:

Network Classifications:

  • PAN (Personal Area Network): Normally used for short-range communications—typically less than a few meters—such as between a wireless headset and a smartphone, or between a wireless mouse and its PC.

  • LAN (Local Area Network): Normally consists of a collection of computers in a single building or building complex. For example, the computers on a university campus or those in a manufacturing plant might be connected by a LAN.

  • MAN (Metropolitan Area Network): Is a network of intermediate size, such as one spanning a local community.

  • WAN (Wide Area Network): Links machines over a greater distance—perhaps in neighboring cities or on opposite sides of the world.

Network Models:

  • Bus Network: Which the machines are all connected to a common communication line.

  • Star Network: One machine serves as a central focal point to which all the others are connected.

Protocols:

  • CSMA/CD (Carrier Sense, Multiple Access with Collision Detection) Ethernet: This protocol dictates that each message be broadcast to all the machines on the bus which is not compatible with wireless star networks in which all machines communicate through a central AP.

  • CSMA/CA (Carrier Sense, Multiple Access with Collision Avoidance) Wifi:

Computer Networks:

  • Repeater: Is little more than a device that passes signals back and forth between the two original buses (usually with some form of amplification) without considering the meaning of the signals.

  • Bridge: It connects two buses, but it does not necessarily pass all messages across the connection but it looks at the destination address that accompanies each message and forwards a message across the connection only when that message is destined for a computer on the other side.

  • Swtich: Is essentially a bridge with multiple connections, allowing it to connect several buses rather than just two.

  • Router: The connection between networks to form an internet provides links between networks while allowing each network to maintain its unique internal characteristics.

  • Gateway: The “point” at which one network is linked to an internet serves as a passageway between the network and the outside world.

Client/Server model:

  • P2P (Peer to Peer): The client/server model involves one process (the server) providing a service to numerous others (clients), the peer-to-peer model involves processes that provide service to and receive service from each other, the peer-to-peer model usually involves processes that execute on a temporary basis.

  • Distributed System: Consist of software units that execute as processes on different computers.

  • Cloud Computing: Describes a distributed system in which many independent computers work closely together to provide computation or services comparable to a much larger machine.

Internet Addressing:

  • Tier 1: ISPs that consist of very high-speed, high-capacity, international WANs. These networks are thought of as the backbone of the Internet. They are typically operated by large companies that are in the communications business.

  • Tier 2: ISPs that tend to be more regional in scope and less potent in their capabilities, these networks tend to be operated by companies in the communications business.

  • Tier 3: An access ISP is essentially an independent Internet, sometimes called an intranet, operated by a single authority that is in the business of supplying Internet access to individual homes and businesses.

IP Adressing:

  • ICANN: Is a nonprofit corporation established to coordinate the Internet’s operation.

  • DNS: Used as an Internet-wide directory system, essentially directories that provide address translation services to clients "translates domain names to IP addresses.".

  • TLD (Top-level Domain): Used to reflect the domain’s classification, TLDs include for commercial institutions, for U.S. government institutions, for nonprofit organizations, for military, etc.

  • Country code TLDS: Two-letter TLDs for each specific country, such as for Australia and for Canada.

Internet Protocols:

  • FTP (File Transfer Protocol): An application for listing and copying files across the network implemented.

  • SSH (Secure Shell): An application for accessing another computer from a great distance

  • HTTP: Traditional network applications

  • SMTP (Simple Mail Transfer Protocol): Defines a way that two computers on the network may interact when transmitting an email message from one host to the other.

World Wide Web:

  • URL: Contains the information needed by a browser to contact the proper server and request the desired document.

  • Client-side activities: The browser would use the information in a URL (perhaps obtained from the person using the browser) to contact the webserver controlling access to the page and ask that a copy of the page be transferred to it.

  • Server-side activities: Respond to client side by sending the text document from anchor tag from code to the browser

Internet four-layer software:

  • Application layer: Responsibility to provide an address that is compatible with the Internet infrastructure, may use the services of the name servers within the Internet to translate mnemonic addresses used by humans into Internet-compatible IP addresses.

  • Transport layer: Is to accept messages from the application layer and to ensure that the messages are properly formatted for transmission over the Internet, also divides long messages into small segments, which are transmitted over the Internet one segment at a time .

  • Network layer: In charge of maintaining the router’s forwarding table and using that table to determine the direction in which to forward packets and which direction a packet should be sent at each step along the packet’s path through the Internet.

  • Link layer: Responsibility of transferring the packet. Thus, the link layer must deal with the communication details particular to the individual network in which the computer resides and hands the packet up to its network layer.

Internet is:

  • Packet-switching network: Is a method of grouping data into packets that are transmitted over a digital network.

  • Fault tolerant: The property that enables a system to continue operating properly in the event of the failure of one or more faults within some of its components.

  • Redundant: A system design in which a component is duplicated so if it fails there will be a backup.

Cybersecurity:

  • Malware: Attacks that incorporate the use of malicious software. Such software might be transferred to, and executed on, the computer itself, or it might attack the computer from a distance.>

  • Virus: Software that infects a computer by inserting itself into programs that already reside in the machine.

  • Worm: An autonomous program that transfers itself through a network, taking up residence in computers and forwarding copies of itself to other computers.

  • Trojan Horse: Program that enters a computer system disguised as a desirable program, such as a game or useful utility package, that is willingly imported by the victim.

  • Spyware: Software that collects information about activities at the computer on which it resides and reports that information back to the instigator of the attack.

  • Phishing: A technique of obtaining information explicitly by simply asking for it.

  • DDoS Attack: A computer in a network can also be attacked by software being executed on other computers in the system.

Protection:

  • Firewall: A primary prevention technique to filter traffic passing through a point in the network, a tool for terminating a denial of service attack because it provides a means of blocking traffic from the attacking computers.

  • Spam Fitler: Are firewalls designed to block unwanted email.

  • Proxy Server: Is a software unit that acts as an intermediary between a client and a server with the goal of shielding the client from adverse actions of the server.


  • Antivirus Software: Used to detect and remove the presence of known viruses and other infections.
  • Cryptography: The study of sending and receiving messages securely in the presence of adversaries.

  • Public key encryption: Involves techniques by which the encryption and decryption algorithms are known, and one of the keys is known, but the encoded message remains secure.
