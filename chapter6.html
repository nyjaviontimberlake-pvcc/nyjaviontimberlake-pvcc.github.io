Operating Systems Terminology

Term Definition
Machine Language Programs for modern computers consist of sequences of instructions that are encoded as numeric digits
Debugging Errors that must be located and corrected before the job is finished.
Assembler Rather than being forced to develop a program directly in machine language, a human could develop a program in mnemonic form and then have it converted into machine language by means of an assembler.
Assembler Language Is inherently machine dependent—that is, the instructions within the program are expressed in terms of a particular machine’s attributes and cannot be easily transported to another computer design because it must be rewritten to conform to the new computer’s register configuration and instruction set.
Translator Translated programs expressed in these high-level primitives into machine-language programs. A translator was similar to the second-generation assemblers.
Compiler Such a translator was similar to the second-generation assemblers, except that it often had to compile several machine instructions into short sequences to simulate the activity requested by a single high-level primitive.
Interpreter These programs were similar to translators except that they executed the instructions as they were translated instead of recording the translated version for future use. Rather than producing a machine-language copy of a program that would be executed later, an interpreter actually executed a program from its high-level form.
Cross-Platform Software Is software that is independent of an operating system’s design as well as the machine’s hardware design and is therefore executable throughout a network.
Natural Language The point was that the programming language was constructed around a small set of primitives that could be expressed in a variety of natural languages with only simple modifications to the translator.
Formal Language A language (a set of strings) that can be described by a finite automaton is called a regular language like Java, C, and Python, are formal languages.
Machine Independence A program written in a third-generation language could theoretically be used on any machine simply by applying the appropriate compiler.
High-Level Programming Language many kinds of abstractions, ranging from the simple—constants, literals, and variables—to moderate complexity—statements, expressions, and control structures—to the esoteric—procedural units, modules, and libraries. Ex. Ada, C, C++, C#, FORTRAN, and Java.
Object-Oriented Programming (OOP) A software system is viewed as a collection of units, called objects, each of which is capable of performing the actions that are immediately related to itself as well as requesting actions of other objects. Together, these objects interact to solve the problem at hand.
Scripting Language A subset of the imperative programming languages is the collection of languages used to perform administrative tasks rather than to develop complex programs.
Data Types Intger: Refers to numeric data consisting of whole numbers, probably stored using two’s complement notation. Operations that can be performed on integer data include the traditional arithmetic operations and comparisons of relative size, such as determining whether one value is greater than another.

Float: Refers to numeric data that might contain values other than whole numbers, probably stored in floating-point notation. Operations performed on data of type float are similar to those performed on data of type integer.

String: Surrounded by either single quotation marks, or double quotation marks. They are arrays of bytes representing unicode characters.
Primitive Data Type The data types that are included as primitives in a programming language, such as for integer and for character.
Data Structure Is the conceptual shape or arrangement of data.
Python Collection Data Structure Are containers used for storing data and are commonly known as data structures, such as lists, tuples, arrays, dictionaries, etc.
Function Is a set of instructions for performing a task that can be used as an abstract tool by other program units.
Local Variable A variable declared within a function, meaning that it can be referenced only within that function.
Global Variable Variables whose scopes are not restricted to a particular part of a program.
Source Program A program in its original form
Object Program The translated version from a source program
